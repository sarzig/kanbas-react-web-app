Cascading Style Sheets (CSS)

Style attribute and tag

Style attribute: The following paragraph achieves its styling via the style attribute:

Style attribute allows configuring look and feel right on the element.

Although it's very convenient it is considered bad practice and you should avoid using the style attribute.

You'll notice that even though the header defines a paragraph style, overwriting the paragraph style on this most local level is the one that eventually renders.

Style tag: The following paragraph achieves its styling via the style tag defined in the header:

A slightly better way to configure look and feel is to declare CSS rules in the STYLE tag in the header and then refer to the tag by their name, ID, or class This paragraph's style is set by a rule referring to the P tags.

NOTE: this styling was commented out so we can instead use labs/A2/index.css

Link tag: The following paragraph achieves its styling via the Link tag defined in the header:

The best way to apply CSS rules is to declare them in a separate CSS file and load it with the LINK tag. Always use this method.

The link tag looks like this:
<link href="index.css" rel="stylesheet"/>
